# AI-Powered Healthcare Shift Fill Prediction

## Project Overview

This project predicts the likelihood of a healthcare shift being filled using a Machine Learning model.

A public recruitment dataset from Kaggle was adapted into a healthcare staffing scenario by creating healthcare-specific features such as profession, shift duration, hourly rate, travel distance, weekend shift, and night shift.

The project includes:
- Data preprocessing
- Feature engineering
- Random Forest classification model
- Model evaluation
- FastAPI REST API

---

## Dataset

Base Dataset:
- aug_train.csv (Kaggle HR Analytics Job Change Dataset)

Generated Dataset:
- healthcare_shift_dataset.csv (Healthcare shift-fill dataset created through feature engineering)

---

## Machine Learning Model

Algorithm:
- Random Forest Classifier

Evaluation Metrics:
- Accuracy: 70.4%
- ROC AUC: 0.747

Additional Evaluation:
- Classification Report
- Confusion Matrix
- Feature Importance

## Setup Instructions

1. Install the required dependencies:

```bash
pip install -r requirements.txt
```

2. Run `project.ipynb` to preprocess the data, train the model, and generate the `shift_fill_model.pkl` file required by the API.

3. Start the FastAPI application:

```bash
uvicorn main:app --reload
```

4. Open the API documentation:

```
http://127.0.0.1:8000/docs
```

## API

The project provides a FastAPI REST API.

### Available Endpoints

- **GET /** – Check whether the API is running.
- **POST /predict** – Predict healthcare shift fill probability and difficulty score.

## Design Decisions

- Used a Random Forest Classifier for prediction because it performs well on structured data.
- Created healthcare-specific features to simulate a healthcare staffing scenario.
- Used FastAPI to build a simple prediction API.

---

## Assumptions

- The generated healthcare dataset represents a simulated healthcare staffing scenario.
- Users provide valid input values to the API.
- The model is generated by running `project.ipynb` before starting the API.

---

## Limitations

- The project uses an adapted public HR dataset instead of real healthcare staffing data.
- Model performance may differ when used with real-world hospital data.
